# 模範解答：API設計

## 認証系

### POST /api/auth/login
**Request**: `{ email: string, password: string }`
**Response**: `{ accessToken: string, refreshToken: string }`
**Error**: 401 Unauthorized

### POST /api/auth/refresh
**Request**: `{ refreshToken: string }`
**Response**: `{ accessToken: string }`
**Error**: 401（トークン無効・期限切れ）

## 予約系

### GET /api/reservations
**Query**: `?status=upcoming|past|cancelled&limit=20&offset=0`
**Response**: `{ reservations: [...], total: number }`
**Auth**: Required（JWT）

### POST /api/reservations
**Request**: `{ facilityId: UUID, startTime: ISO8601, endTime: ISO8601, purpose?: string }`
**Response**: `201 Created`
**Validation**:
- `startTime < endTime`
- 施設の営業時間内か
- 既存予約と重複しないか

### PATCH /api/reservations/[id]
**Request**: `{ startTime?: ISO8601, endTime?: ISO8601, purpose?: string }`
**制約**: 予約開始時刻前のみ
**Response**: `200 OK`

### POST /api/reservations/[id]/cancel
**Auth**: Required（予約者本人）
**Response**: `{ id, status: "cancelled", cancelledAt: ISO8601 }`
