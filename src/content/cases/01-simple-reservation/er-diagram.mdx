# DB設計・ER図

```mermaid
erDiagram
    USER ||--o{ RESERVATION : makes
    FACILITY ||--o{ RESERVATION : booked_by
    
    USER {
        uuid id PK
        string email UK
        string password_hash
        string name
        datetime created_at
        datetime updated_at
    }
    
    FACILITY {
        uuid id PK
        string name
        string description
        int capacity
        string image_url
        datetime created_at
    }
    
    RESERVATION {
        uuid id PK
        uuid user_id FK
        uuid facility_id FK
        datetime start_time
        datetime end_time
        string purpose
        enum status "confirmed|cancelled"
        datetime created_at
        datetime updated_at
    }
```

## 設計ポイント
- `RESERVATION.status` は enum（confirmed / cancelled）。物理削除は行わない（監査・履歴のため）
- `USER.password_hash` は平文パスワードを絶対に保存しない
- `RESERVATION` に複合ユニーク制約を検討：`facility_id + start_time` の重複をDBレベルで防ぐ。ただし、キャンセル済みは除く必要があるため、部分インデックス（PostgreSQL）を使う
