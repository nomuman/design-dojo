# API設計

## 認証系

### POST /api/auth/login
- **Request**: `{ email: string, password: string }`
- **Response**: `{ accessToken: string }`（refreshToken は HttpOnly Cookie で返却）
- **Error**: 401（認証失敗）
- **備考**: refreshToken を localStorage に保存すると XSS 攻撃を受けるリスクがあるため、必ず HttpOnly Cookie で管理すること

### POST /api/auth/refresh
- **Request**: Cookie から refreshToken を読み取る（Body ではない）
- **Response**: `{ accessToken: string }`
- **Error**: 401（トークン無効・期限切れ）

## 予約系

### GET /api/reservations
- **Query**: `?status=upcoming|past|cancelled&limit=20&offset=0`
- **Response**: `{ reservations: Reservation[], total: number }`
- **Auth**: Required（自分の予約のみ）

### POST /api/reservations
- **Request**: `{ facilityId: UUID, startTime: ISO8601, endTime: ISO8601, purpose?: string }`
- **Response**: `201 Created` → `Reservation`
- **Validation**:
  - `startTime < endTime`
  - 施設の営業時間内か
  - 既存予約と時間帯が重複しないか（DB制約 + アプリケーション層の二重チェック）

### PATCH /api/reservations/[id]
- **Request**: `{ startTime?: ISO8601, endTime?: ISO8601, purpose?: string }`
- **制約**: 予約開始時刻前のみ変更可能
- **Response**: `200 OK` → `Reservation`

### POST /api/reservations/[id]/cancel
- **Auth**: Required（予約者本人のみ）
- **Response**: `200 OK` → `{ id, status: "cancelled", cancelledAt: ISO8601 }`

## エラーレスポンス統一フォーマット
